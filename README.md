## Requirements
Some components will not style correctly without this.

- Requires [Stylus Shadow DOM Support](https://greasyfork.org/en/scripts/424030-stylus-shadow-dom-support) by jcunews.
  > You need a userscript manager like [ViolentMonkey](https://violentmonkey.github.io/#installation) to complete the installation.


- Browser must support [Baseline 2023](https://web.dev/baseline/2023) features.

### Known Limitations
- Most common UI elements are covered, though some niche components may still need work. Contributions, issues, and suggestions are welcome.
- Some elements may briefly appear unstyled before the script injects the theme. This is a limitation of **Stylus Shadow DOM Support**.
- CSS improvements are ongoing. Portions of the theme will continue to be cleaned up and reorganized.
- Designed for Carleton but has partial compatibility for other institutions that also use D2L Brightspace.

## Contributing
You may use this project as a template for your own D2L themes. If you're making a dark theme for your institution, feel free to create a Pull Request while you're at it. Contributions, issues, and suggestions are welcome.

### Mission Statement
This project aims to provide a cohesive, dark look across Brightspace. Unlike automatic dark-theme plugins, which often produce unappealing artifacts or unreadable elements, this theme offers a consistent style with additional clarity and contrast.
